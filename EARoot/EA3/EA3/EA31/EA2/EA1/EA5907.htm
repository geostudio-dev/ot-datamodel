The LC_HorizontalPattern object allows the ordering of LC_Stratum objects consisting of single or groupings of LC_Elements, into one or more horizontal pattern(s). A horizontal pattern may be used for a complex Land Cover object composed by two or more distinct land cover features that will be handled as a “unicum” independently from scale constrains. An example is the vegetation formation called "tiger bush" which consists of a combination of patches of open shrubs and patches of open grassland in a specific horizontal pattern. These two LC_Element objects are treated together as one land cover object, independent of scale. An example of tiger bush is given in C.2.  Using the horizontal pattern construct a user can indicate that at any scale this object will be always a combination of these two LC_Element objects.<br/><br/>NOTE A horizontal pattern for a object is distinct from the combination of two LC_LandCover objects that would otherwise be distinct but are combined because they can not be distinguished at a given scale. For example closed trees (class A) in a grassland (class B) area that due to the scale of representation of the data can not be mapped separately and are therefore combined into a combined class A/B. At other scales these would be separate land cover classes. <br/><br/>The object LC_HorizontalPattern has three optional attributes <i>patternType</i>, <i>patternCoveragePercentage</i> and <i>patternOccurance</i>. The permitted values of the attribute <i>patternType</i> are described in free text in a character string. The permitted values of the attribute <i>patternCoveragePercentage</i> are described by the values object LC_PermittedPercentageValue. The permitted values of the attribute <i>patternOccurance</i> are described by the values object LC_PermittedPercentageValue. <br/><br/>
